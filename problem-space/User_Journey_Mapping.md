# User Journey Mapping

## Kurzbeschreibung
User Journey Mapping visualisiert die komplette Erfahrung eines Nutzers mit einem Produkt oder Service über alle Berührungspunkte hinweg. Die Methode hilft dabei, Schmerzpunkte zu identifizieren und die Nutzererfahrung ganzheitlich zu verbessern.

## Einsatzzweck
Diese Methode wird eingesetzt, um bestehende Nutzererfahrungen zu analysieren, Verbesserungspotentiale zu identifizieren und ein gemeinsames Verständnis für die Customer Experience im Team zu schaffen. Besonders wertvoll in der Analyse- und Konzeptionsphase von Produktentwicklungsprojekten.

## Kurzanleitung

1. **Persona definieren:** Wähle eine spezifische Nutzergruppe aus und erstelle eine detaillierte Persona → "Anna, 28, Freelance-Grafikerin, technikaffin"

2. **Szenario festlegen:** Bestimme ein konkretes Nutzungsszenario mit klarem Ziel → "Anna möchte ein neues Design-Tool für ihr nächstes Kundenprojekt finden"

3. **Touchpoints sammeln:** Liste alle Berührungspunkte chronologisch auf → Websuche, Website-Besuch, Registrierung, erste Nutzung, Support-Kontakt

4. **Journey-Phasen strukturieren:** Gliedere die Reise in logische Abschnitte → Awareness, Consideration, Purchase, Onboarding, Usage, Support

5. **Nutzeraktionen dokumentieren:** Beschreibe was der Nutzer in jeder Phase tut → "Vergleicht Preise verschiedener Tools", "Lädt kostenlose Testversion herunter"

6. **Emotionen erfassen:** Bewerte die emotionale Verfassung des Nutzers → Neugier (+), Frustration (-), Begeisterung (++)

7. **Pain Points identifizieren:** Markiere Problembereiche und Reibungspunkte → Lange Ladezeiten, unklare Navigation, fehlende Hilfestellungen

8. **Verbesserungschancen ableiten:** Entwickle konkrete Lösungsansätze für identifizierte Probleme → Schnellere Server, intuitivere Benutzerführung, Video-Tutorials

## Beispielprompt
{% code overflow="wrap" %}
```
Erstelle eine User Journey Map für einen Online-Shop. Nutzer-Persona: "Michael, 35, kauft Laufschuhe für seinen ersten Marathon". Berücksichtige die Phasen: Bedürfnis, Recherche, Vergleich, Kauf, Lieferung, erste Nutzung. Identifiziere mögliche Pain Points und emotionale Höhen/Tiefen.
```
