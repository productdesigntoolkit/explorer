# User Story Mapping

## Kurzbeschreibung
User Story Mapping ist eine visuelle Technik, um User Stories in einem zweidimensionalen Raster zu organisieren und zu priorisieren. Die Methode hilft dabei, ein gemeinsames Verständnis der User Journey zu entwickeln und Features nach Wert und Priorität zu strukturieren.

## Einsatzzweck
User Story Mapping wird eingesetzt, um komplexe Produktanforderungen zu strukturieren, ein Backlog zu erstellen und Release-Pläne zu entwickeln. Die Methode eignet sich besonders gut für agile Teams, die eine nutzerzentrierte Sicht auf ihr Produkt entwickeln möchten.

## Kurzanleitung
1. **Backbone erstellen:** Definiere die wichtigsten User Activities horizontal als Spalten → Registrieren, Profil erstellen, Suchen, Buchen
2. **User Stories sammeln:** Schreibe alle relevanten User Stories auf Haftnotizen im Format "Als [Nutzer] möchte ich [Ziel], damit [Nutzen]"
3. **Stories zuordnen:** Ordne jede User Story der passenden User Activity zu und platziere sie vertikal unter der entsprechenden Spalte
4. **Nach Priorität sortieren:** Sortiere die Stories in jeder Spalte von oben nach unten nach Wichtigkeit und Reihenfolge im Nutzungsprozess
5. **Walking Skeleton identifizieren:** Markiere die oberste Reihe als MVP (Minimum Viable Product) mit den essentiellen Stories
6. **Releases definieren:** Ziehe horizontale Linien, um verschiedene Release-Zyklen zu markieren und Stories entsprechend zu gruppieren
7. **Lücken identifizieren:** Prüfe die Map auf fehlende Stories oder Inkonsistenzen in der User Journey
8. **Diskutieren und verfeinern:** Führe Teamdiskussionen über Prioritäten, Abhängigkeiten und Scope-Entscheidungen
