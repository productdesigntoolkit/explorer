# Mockup Method

## Kurzbeschreibung
Die Mockup Method erstellt statische, visuelle Prototypen von digitalen oder physischen Produkten, um Design-Konzepte schnell zu visualisieren und zu validieren. Sie ermöglicht es, das Aussehen und die Benutzererfahrung zu testen, ohne funktionale Prototypen entwickeln zu müssen.

## Einsatzzweck
Diese Methode wird in frühen Design-Phasen eingesetzt, um Ideen visuell zu kommunizieren, Stakeholder-Feedback einzuholen und Design-Entscheidungen zu treffen. Besonders wertvoll für die Präsentation von User Interfaces, Produktverpackungen oder physischen Produkten vor der aufwändigen Entwicklung.

## Kurzanleitung
1. **Zielgruppe definieren:** Bestimme, wer das Mockup betrachten wird und welche Informationen vermittelt werden sollen
2. **Fidelität festlegen:** Entscheide zwischen Low-Fidelity (grob, schnell) oder High-Fidelity (detailliert, realistisch) basierend auf dem Verwendungszweck
3. **Medium wählen:** Wähle das passende Tool → Papier, Figma, Sketch, Photoshop oder physische Materialien
4. **Content strukturieren:** Sammle alle notwendigen Inhalte wie Texte, Bilder, Icons und organisiere sie logisch
5. **Layout erstellen:** Gestalte die grundlegende Struktur und platziere Elemente entsprechend der Informationshierarchie
6. **Visuelle Details hinzufügen:** Ergänze Farben, Typografie, Spacing und andere Design-Elemente für Realitätsnähe
7. **Kontext simulieren:** Präsentiere das Mockup in realistischen Umgebungen → Smartphone-Screen, Produktregal, Anwendungssituation
8. **Feedback sammeln:** Zeige das Mockup Testpersonen und dokumentiere Reaktionen und Verbesserungsvorschläge

## Beispielprompt
{% code overflow="wrap" %}
```
Erstelle ein High-Fidelity Mockup für eine Fitness-App Startseite. Zeige die wichtigsten Features wie Workout-Tracking, Fortschrittsanzeige und soziale Funktionen. Präsentiere es auf einem iPhone-Screen mit realistischem Kontext.
```
