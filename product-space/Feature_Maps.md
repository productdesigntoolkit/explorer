# Feature Maps

## Kurzbeschreibung
Feature Maps visualisieren alle Funktionen eines Produkts in einer strukturierten Übersicht und zeigen deren Zusammenhänge sowie Prioritäten auf. Sie helfen dabei, den Funktionsumfang zu definieren und strategische Entscheidungen über die Produktentwicklung zu treffen.

## Einsatzzweck
Feature Maps werden in der Konzeptionsphase eingesetzt, um den Funktionsumfang eines Produkts zu planen und zu priorisieren. Sie eignen sich besonders für komplexe digitale Produkte, bei denen verschiedene Features miteinander interagieren und eine klare Roadmap benötigt wird.

## Kurzanleitung
1. **Nutzergruppen definieren:** Identifiziere die wichtigsten Zielgruppen und deren Bedürfnisse → Endkunden, Admins, Partner
2. **Features sammeln:** Liste alle möglichen Funktionen auf, die das Produkt haben könnte → Login, Profil, Suche, Bezahlung
3. **Features kategorisieren:** Gruppiere ähnliche Funktionen in thematische Bereiche → Nutzerverwaltung, Content-Management, Commerce
4. **Abhängigkeiten identifizieren:** Markiere, welche Features voneinander abhängig sind → Warenkorb benötigt Produktkatalog
5. **Prioritäten festlegen:** Bewerte Features nach Wichtigkeit und Aufwand → Must-have, Should-have, Could-have
6. **Timeline zuordnen:** Ordne Features verschiedenen Entwicklungsphasen zu → MVP, Version 1.1, Future Releases
7. **Visualisierung erstellen:** Erstelle eine übersichtliche Karte mit Kategorien, Prioritäten und Abhängigkeiten
8. **Stakeholder-Review:** Validiere die Feature Map mit dem Entwicklungsteam und den Stakeholdern

## Beispielprompt
{% code overflow="wrap" %}
```
Erstelle eine Feature Map für eine E-Learning-Plattform. Berücksictige dabei drei Nutzergruppen: Lernende, Lehrende und Administratoren. Kategorisiere die Features nach Bereichen wie Kursverwaltung, Kommunikation und Bewertung. Priorisiere nach Must-have für den MVP und Nice-to-have für spätere Versionen.
```
{% endcode %}
