# Tech Stack Selection Matrix

## Kurzbeschreibung
Eine systematische Bewertungsmatrix zur objektiven Auswahl der optimalen Technologien für ein Digitalprojekt. Sie hilft dabei, verschiedene Tech-Optionen anhand definierter Kriterien zu vergleichen und datenbasierte Entscheidungen zu treffen.

## Einsatzzweck
Diese Methode eignet sich besonders in der frühen Projektphase, wenn grundlegende Technologie-Entscheidungen getroffen werden müssen. Sie ist ideal für interdisziplinäre Teams, die gemeinsam eine fundierte Tech-Stack-Auswahl treffen wollen, ohne sich von subjektiven Präferenzen leiten zu lassen.

## Kurzanleitung

1. **Kriterien definieren:** Bestimme 5-8 relevante Bewertungskriterien wie Performance, Lernkurve, Community-Support, Kosten, Skalierbarkeit oder Wartungsaufwand.

2. **Gewichtung festlegen:** Vergib für jedes Kriterium eine Gewichtung von 1-5 basierend auf der Projektpriorität → Performance könnte 5 sein, Community-Support nur 2.

3. **Tech-Optionen sammeln:** Liste alle ernsthaft in Betracht kommenden Technologien auf → React vs. Vue.js vs. Angular für Frontend oder Node.js vs. Python vs. Java für Backend.

4. **Matrix erstellen:** Erstelle eine Tabelle mit Technologien als Zeilen und Kriterien als Spalten, inklusive einer Spalte für den gewichteten Gesamtscore.

5. **Einzelbewertung:** Bewerte jede Technologie für jedes Kriterium auf einer Skala von 1-5 Punkten → React bekommt bei "Community-Support" eine 5, bei "Lernkurve" vielleicht eine 3.

6. **Gewichtete Scores berechnen:** Multipliziere jeden Einzelscore mit der entsprechenden Kriteriengewichtung und addiere alle gewichteten Werte pro Technologie.

7. **Ranking erstellen:** Sortiere die Technologien nach ihrem Gesamtscore in absteigender Reihenfolge zur Entscheidungsunterstützung.

8. **Ergebnis validieren:** Überprüfe das Ranking mit dem Team und berücksichtige eventuelle qualitative Faktoren, die in der Matrix nicht abgebildet wurden.

## Beispielprompt

{% code overflow="wrap" %}
```
Erstelle eine Tech Stack Selection Matrix für ein E-Commerce-Projekt mit folgenden Kriterien: Performance (Gewichtung 5), Entwicklungsgeschwindigkeit (4), Community-Support (3), Kosten (4), Skalierbarkeit (5). Bewerte die Frontend-Optionen React, Vue.js und Angular auf einer Skala von 1-5.
```
